Description

Right-to-Left (RTL) support represents the ability of a library, website, or application to handle and respond to users who communicate through right-to-left languages. Right-to-left languages are Arabic, Hebrew, Chinese, Japanese, and others.

The default language input for web users can be left-to-right or right-to-left. The Telerik UI for ASP.NET Core PanelBar supports both input modes enabling you to properly render a spectrum of languages, including Hebrew, Chinese, and Arabic, thus making your app a truly global one and helping your localization efforts.

The Telerik UI for ASP.NET Core PanelBar component comes with built-in right-to-left (RTL) support and offers right-to-left rendering of all elements and items, which can be configured by wrapping the component configuration in a div element with k-rtl class.
