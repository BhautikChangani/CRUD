Description

Keyboard navigation provides all web and app users with a fast keyboard-only navigation capability and is also part of the web accessibility featuresâ€”It enables users with disabilities to fully control or access their website application through the keyboard.

By default, users can only navigate to links, buttons, and form controls with a keyboard. The navigation order in which interactive items receive keyboard focus has to be logical and intuitive. Generally, keyboard navigation logic needs to follow the visual horizontal and vertical flow of the page, for example, left to right and top to bottom, header first followed by the main and then the page navigation.

The Telerik UI for ASP.NET MVC ToolBar is always available as demonstrated in this demo.

The ToolBar supports the following keyboard shortcuts:

Focus

  • Alt + W Focuses the component.

Supported Keys and User Actions

  • If .NavigateOnTab(true)
    Tab Focuses the next focusable item.
  • Shift + Tab Focuses the previous focusable item.
  • (Default) If .NavigateOnTab(false)
    Right Arrow Focuses the next focusable item.
  • Left Arrow Focuses the previous focusable item.
  • General
    Enter Selects the focused button, or enters the inner template navigation if the template contains focusable items.
  • Space Selects the focused button.
  • Home Focuses the first focusable item.
  • End Focuses the last focusable item.
  • Esc If the inner template navigation is activated, moves the focus back to the ToolBar item.

Actions Applied to the SplitButton or CommandOverflow

  • Alt + Down Opens the menu of the SplitButton or ComandOverflow.
  • Down Arrow Focuses the next menu item of the SplitButton or CommandOverflow.
  • Up Arrow Focuses the previous menu item of the SplitButton or CommandOverflow.
  • Alt + Up Arrow Closes the menu of the SplitButton or ComandOverflow.
  • Esc Closes the menu of the SplitButton or ComandOverflow.
  • Enter Selects the focused button.
  • Space Selects the focused button.
