Description

Keyboard navigation provides all web and app users with a fast keyboard-only navigation capability and is also part of the web accessibility features—it enables users with disabilities to fully control their website or app access through the keyboard.

By default, users can only navigate to links, buttons, and form controls with a keyboard. The navigation order in which interactive items receive keyboard focus has to be logical and intuitive. Generally, keyboard navigation logic needs to follow the visual horizontal and vertical flow of the page, for example, left to right and top to bottom, header first followed by the main and then the page navigation.

The keyboard navigation of the Telerik UI for ASP.NET MVC Menu is always available, as demonstrated in this demo.

The Menu supports the following keyboard shortcuts:

Focus

  • Alt + W Focuses the vertical menu (clicking on it or tabbing also work).
  • Alt + Q Focuses the horizontal menu (clicking on it or tabbing also work).

Supported Keys and User Actions

  • Home Goes to the first item.
  • End Goes to the last item.
  • Right Goes to the next item or opens an item group.
  • Left Goes to the previous item or closes an item group.
  • Down Opens an item group or goes to the next item in a group.
  • Up Goes to the previous item in a group.
  • Enter Selects or navigates the item (same as click).
  • Space Selects or navigates the item (same as click).
  • Esc Closes the innermost opened group.
  • Tab Tabs away from the Menu to the next focusable page element.
  • Shift + Tab Tabs away from the Menu to the previous focusable page element.
