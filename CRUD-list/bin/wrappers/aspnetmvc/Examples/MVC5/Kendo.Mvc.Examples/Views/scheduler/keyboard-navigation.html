Description

Keyboard navigation provides all web and app users with a fast keyboard-only navigation capability and is also part of the web accessibility features--it enables users with disabilities to fully control their website or app access through the keyboard.

By default, users can only navigate to links, buttons, and form controls with a keyboard. The navigation order in which interactive items receive keyboard focus has to be logical and intuitive. Generally, keyboard navigation logic needs to follow the visual horizontal and vertical flow of the page, for example, left to right and top to bottom, header first followed by the main and then the page navigation.

Keyboard navigation is always available in the Telerik UI for ASP.NET MVC Scheduler as demonstrated in this demo.

Focus

  • Alt + W Focuses the component.

Actions Applied to the Entire Scheduler

  • F10 Focuses the toolbar.
  • B (Available in the Day, Week, and Timeline views) Toggles the view between business hours and full day.
  • C Opens the Create new event pop-up editor by using the start date of the current view as a start and end.
  • T Navigates the view to today's time period.
  • Alt + 1 - 5 Switch between the available views.
  • Shift + Right Arrow Navigates to the next time period.
  • Shift + Left Arrow Navigates to the previous time period.

Actions Applied to the Toolbar of the Scheduler

  • Right Arrow Focuses the next toolbar item.
  • Left Arrow Focuses the previous toolbar item.
  • Enter or Space Activates the toolbar item.

Actions Applied to the Events of the Scheduler

  • Right Arrow or Down Arrow Focuses the next available event.
  • Left Arrow or Up Arrow Focuses the previous available event.
  • Enter Opens the Edit window for the selected event.
  • Esc Closes the Edit popup window.
  • Delete or Backspace Opens the Delete confirmation popup to delete the event.
