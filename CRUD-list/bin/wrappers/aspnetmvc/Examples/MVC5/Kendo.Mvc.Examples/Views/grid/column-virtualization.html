Description

The Telerik UI for ASP.NET MVC Data Grid provides the column-virtualization feature, which enables you to operate with a large number of columns where displaying them at once will not impose a performance penalty due to limited browser resources.

This demo shows how to implement the column-virtualization functionality of the ASP.NET MVC Data Grid to mitigate any slowdowns when operating with huge numbers of columns. To enable column virtualization, use the .Scrollable(scrollable => scrollable.Virtual(GridVirtualizationMode.Columns)) configuration option. As a result, the component will display only part of all its columns, and when the user scrolls the Grid, the visual subset of columns will change accordingly.

You can also use column virtualization together with row virtualization and, in this way, increase the performance substantially of the Data Grid.

For more information about the column-resizing feature of the component, refer to the official Telerik UI for ASP.NET MVC Data Grid product documentation on column settings.
