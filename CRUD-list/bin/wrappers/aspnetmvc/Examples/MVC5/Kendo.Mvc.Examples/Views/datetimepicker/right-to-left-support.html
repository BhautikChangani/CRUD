Description

Right-to-Left (RTL) support represents the ability of a library, website, or application to handle and respond to users who communicate through right-to-left languages. Right-to-left languages are Arabic, Hebrew, Chinese, Japanese, and others.

The default language input for web users can be left-to-right or right-to-left. The Telerik UI for ASP.NET MVC DateTimePicker component supports both input modes enabling you to properly render a spectrum of languages, including Hebrew, Chinese, and Arabic, thus making your app a truly global one and helping your localization efforts.

This demo showcases how to set the DateTimePicker Right-to-Left (RTL) functionality by specifying k-rtl in the div class.
