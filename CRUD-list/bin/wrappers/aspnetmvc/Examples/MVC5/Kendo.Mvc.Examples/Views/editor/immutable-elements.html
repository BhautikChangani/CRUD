Description

The Telerik UI for ASP.NET MVC Editor and its immutable elements feature enable you to predefine elements in the content area that cannot be edited by the end user.

This demo shows a use case where a mail form will be edited. You can see also how to implement the following actions:

  • Enable the immutable elements by using the Immutables option.
  • Define the HTML elements that cannot be edited by adding the contenteditable='false' attribute.
  • Use the Immutables.Serialization and Immutables.Deserialization option to change the data shown in the viewHtml dialog and enable the end user to control the CSS text-align property from there.
  • Define the default styles for the immutable elements.
  • To improve UX when an immutable element is selected by defining the styles.
