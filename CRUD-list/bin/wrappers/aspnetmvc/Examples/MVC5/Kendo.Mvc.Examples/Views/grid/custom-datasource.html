Description

The Telerik UI for ASP.NET MVC Data Grid enables you to bypass the built-in data processing and handle operations such paging, sorting, filtering, and grouping yourself.

This demo shows how to bind the Data Grid to a custom data source when none of the built-in types are suitable. To make the connection to the remote endpoint and process the data properly, the data source requires information about the web service URLs, the request type, the response data type, and the structure (schema) of the response.

For more details about the service, refer to the Kendo UI Demos Service repository on GitHub.

For more information about the data-binding capabilities, refer to the official Telerik UI for ASP.NET MVC Data Grid product documentation on the available data-binding approaches.
